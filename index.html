DNS 服务器

发表于 2020-10-31 分类于 网络安全 ， Windows

DNS 服务器

  1. DNS（Domain Name System，域名系统），一个域名与 IP 地址相互映射的分布式数据库；
  2. DNS 服务器为客户机提供域名解析服务，通过域名最终得到域名对应 IP 地址的过程叫做域名解析；
  3. 主机名.域名称为完全限定域名（FQDN）；
  4. 一个域名下可以有多个主机，域名是全球唯一的，所以主机名.域名也是全球唯一的；
  5. DNS 服务器的分类：
    1. 根名称服务器；
    2. 主要名称服务器；
    3. 辅助名称服务器；
    4. 高速缓存名称服务器；
  6. 常见的域名解析类型：
    1. A 记录：正向解析；
    2. CNAME 记录：别名解析；
    3. PTR 记录：反向解析；
    4. MX 记录：邮件交换记录解析；
    5. NS 记录：域名服务器解析；
  7. DNS 解析按照查询方式分为两种：
    1. 递归查询：客户机与本地 DNS 服务器之间的解析过程，即，所问即所答；
    2. 迭代查询：本地 DNS 服务器与根或其他 DNS 服务器之间的解析过程，即，只告诉你线索，答案我不知道；
  8. DNS 转发的过程是递归查询；
  9. DNS 解析按照查询内容分为两种：
    1. 正向解析：已知域名，解析 IP 地址；
    2. 反向解析：已知 IP 地址，解析域名；
  10. 客户机请求域名解析的顺序：
    1. 本机 DNS 缓存；
    2. 本机 Host 文件；
    3. 本地 DNS 服务器；
  11. DNS 服务器对域名请求的解析顺序：
    1. DNS 高速缓存；
    2. 本地区域解析文件；
    3. 转发器；
    4. 根；
  12. DNS 服务的端口是TCP/UDP:53；
  13. nslookup 域名：查看指定域名的 DNS 解析；
  14. ipconfig /flushdns：清除 DNS 缓存；
  15. ipconfig /displaydns：查看本机 DNS 缓存；

用户与组管理

发表于 2020-10-18 分类于 网络安全 ， Windows

用户管理

  1. administrator 是 Windows 自带的管理员账户，UID 固定是 500；

  2. guest 是 Windows 自带的来宾账户，权限低于普通账户；

  3. 普通用户的 UID 从 1000 开始；

  4. 每个用户都有自己的配置文件，在用户第 1 次登录时自动生成；

  5. Windows 账户和密码的存储位置是：c:\windows\system32\config\SAM，可以暴力、撞库，或者通过 PE 替换哈希值；

  6. Windows 服务器默认密码的最长有效期是 42 天；

DOS 命令

  1. echo：输出文本或变量的值；

  2. echo > xxx：echo 配合重定向符号 > 可以将文本写入到指定的文件中，如果文件不存在则创建；

  3. echo >> xxx: echo 配合重定向符号 >> 可以将文本追加到指定文件的末尾，如果文件不存在则创建；

  4. type：显示指定文件的内容，文件内容过长时，可配合管道符号 | 及参数 more 分页显示，如 type test.txt | more；

  5. del：删除文件，可使用正则表达式批量删除；

IP 地址

  1. 路由器用户连接外网和内网；
  2. 交换机用于组建局域网（内网）；
